Section 02

The 4 Input Variables

Brand DNA 파이프라인에 진입하는 4가지 독립 변수입니다. 각 변수의 역할, 영향력, 제어 범위를 정의합니다.

Brand Preset JSON ~50%

프롬프트 전체의 뼈대 — 포즈, 표정, 배경, 톤, 컨셉 기본값을 결정합니다.

Files db/brand_presets/*.json
Examples mlb.json, discovery.json, stretch_angels.json

Determines

배경 세팅 포즈 기본값 표정 무드 색감 톤 한국어 프롬프트 텍스트
// mlb.json (excerpt) { "brand": "MLB", "background": { "장소": "도시 야외", "배경상세": "콘크리트 벽, 그래피티" }, "pose": { "stance": "wide_stance", "vibe": "confident, street" }, "korean_text": "파워포즈, 쿨톤..." }
DNA Profile ~15%

Preset 위에 미세 조정 — VLM으로 레퍼런스 분석한 결과를 기반으로 일부 필드를 패치합니다.

Data dna_profiles table
Fields profile_json, centroid_vector (30d)

Determines

stance arm position 표정 베이스 vibe keywords
Limitation: ~20개 이상의 프롬프트 필드 중 약 6개만 패치합니다. 나머지는 Brand Preset 기본값이 그대로 유지됩니다.
Image Inputs ~30%

인물과 착장을 결정 — 결과 이미지의 "누가 뭘 입고 있는지"를 정의합니다.

Types Face image, Outfit image(s)
Format PNG/JPG, 최대 4장 outfit 이미지

Determines

얼굴 동일성 착장 정확도 VLM 착장 분석
Note: 포즈, 배경, 표정에는 영향을 주지 않습니다. 이미지가 직접 제어하는 것은 인물의 아이덴티티와 착장뿐입니다.
Background Reference ~5%

배경 스타일 제어 (선택적) — 제공 시 배경 환경을 오버라이드합니다.

Modes text_only | loose | strict
Default Preset 배경 사용 (미제공 시)

Determines

배경 환경 배경 스타일 참조도
Issue: 배경 정보가 3곳에 분산되어 있습니다 — preset + VLM 분석 + 이미지. 이로 인해 충돌 가능성이 존재합니다.
Section 03

Influence Proportion

최종 결과 이미지에 대한 각 인풋 변수의 영향력 비율입니다. Brand Preset이 파이프라인의 지배적 변수임을 확인할 수 있습니다.

Brand Preset JSON 50%

프롬프트의 골격을 결정합니다. 포즈, 표정, 배경, 톤, 네거티브, 한국어 텍스트까지 거의 모든 프롬프트 필드의 기본값이 여기서 나옵니다.

Image Inputs 30%

"누가" "뭘 입고" 있는지를 결정합니다. Gemini가 직접 참조하는 이미지이므로 텍스트 프롬프트와 별개로 강한 영향력을 가집니다.

DNA Profile 15%

Preset 위에 미세 조정하지만, ~20개 필드 중 약 6개만 패치합니다. 나머지는 Preset 기본값이 그대로 유지되어 영향력이 제한적입니다.

Background Ref 5%

선택적 입력이며, 배경 한 가지만 제어합니다. 미제공 시 Preset 배경이 사용되므로 전체 영향력은 낮습니다.

Section 04

Step-by-Step Input Flow

각 처리 단계에서 어떤 인풋이 진입하고, 어떤 아웃풋이 생성되는지 순서대로 추적합니다.

STEP 01
build_prompt()
Input Brand Preset JSON + VLM 착장분석 결과
Output 전체 프롬프트 JSON (포즈, 표정, 배경, 착장, 모델 등)
이 시점에서 _korean_prompt도 생성됩니다 (preset 텍스트 기반). VLM 착장 분석 결과가 outfit 필드에 반영됩니다.
STEP 02
VLM Analysis
Input Outfit image → 착장 분석  |  Background image → 배경 분석
Output 착장 설명 텍스트, 배경 설명 텍스트 (user_options에 전달)
배경 분석은 text_only 모드에서만 프롬프트에 반영됩니다. loose/strict 모드에서는 이미지 자체가 참조됩니다.
STEP 03
adapt_prompt()
Input DNA Profile (pose_distribution, expression_distribution, top_vibe_keywords)
Output 일부 필드 패치된 프롬프트 JSON

Patched Fields

stance 왼팔 (left_arm) 오른팔 (right_arm) 표정.베이스 표정.입 표정.시선 주제.mood 다리 (legs) 힙 (hips) 눈썹 (eyebrows) 배경 (background) 톤 (tone) 네거티브 (negative) 모든 텍스트
STEP 04
Korean Prompt Regeneration
Input Adapted JSON + _dna_brand field
Output 재생성된 _korean_prompt
Preset의 korean_text에서 브랜드별 동적 텍스트를 로드합니다.
MLB → "파워포즈, 쿨톤"  |  Discovery → "자연스러운 포즈, 자연광"
STEP 05
generate_brandcut()
Input 최종 프롬프트 + Face/Outfit/Background 이미지
Output 생성 이미지 (Gemini API)

Gemini API Parts 순서

1. Pose ref 2. Korean prompt 3. JSON 4. Style ref 5. Background ref 6. Pose/Expression 7. Outfit details 8. Face images 9. Outfit images

Section 06

향후 발전 방향

DNA 시스템의 핵심 의사결정 과제

Brand DNA 파이프라인이 더 정교하게 브랜드 톤을 반영하려면, 아래 핵심 과제들에 대한 의사결정이 필요합니다.

6.1 — DNA 카테고리 정의

현재 DNA는 3개 카테고리(pose, expression, vibe)에서 30개 차원만 추출합니다. 더 많은 카테고리를 추가해야 할까요?

Category 현재 포함 Potential Value
Pose (포즈) ✓ arm position, stance, energy level
Expression (표정) ✓ mouth, eyes, intensity
Vibe (분위기) ✓ keywords like cool, chic, etc.
Background (배경) ❌ urban / nature / studio preference from references
Composition (구도) ❌ camera angle, framing, depth of field
Styling (스타일링) ❌ fit preference, layering style, color palette
Mood/Lighting (무드/조명) ❌ warm / cool tone, golden hour, studio light
Accessories (악세서리) ❌ earrings, sunglasses, bags preference

Trade-off

카테고리가 늘어날수록 DNA가 더 정밀해지지만, 더 많은 레퍼런스 이미지가 필요하고 추출 시간이 길어집니다. 현재 3장 레퍼런스 × 3 카테고리 구조에서, 8 카테고리로 확장하면 정확한 추출을 위해 최소 5~8장의 레퍼런스가 필요할 수 있습니다.

6.2 — DNA Profile vs Brand Preset 충돌 해결

DNA Profile과 Brand Preset이 충돌할 때, 어느 쪽이 우선하는가? 이것이 시스템의 핵심 설계 결정(KEY decision point)입니다.

Brand Preset (mlb.json)
표정.베이스 = "fierce"
stance = "power stance"
배경 = "콘크리트"
vs
DNA Profile (from 3 references)
표정.베이스 → "natural" (inferred from vibe)
stance = "relaxed"
배경 = (not extracted)

현재 동작: DNA가 패치하는 필드는 DNA가 우선하고, 나머지는 Preset이 담당합니다. 이 균형이 올바른 것인가?

6.3 — 변수 간 독립성과 연결

4가지 인풋 변수 간의 상호 참조 관계를 분석합니다. 현재 구조에서 모든 변수는 완전히 독립적입니다. 연결해야 할까요?

Preset DNA Image Bg Ref
Preset — ❌ ❌ ❌
DNA ❌ — ❌ ❌
Image ❌ ❌ — ❌
Bg Ref ❌ ❌ ❌ —

장점: 모듈성

각 변수를 독립적으로 변경할 수 있습니다. Preset을 바꿔도 DNA Profile에 영향이 없고, 이미지를 교체해도 프롬프트 구조는 유지됩니다.

장점: 테스트 용이성

단일 변수만 변경하고 나머지를 고정하여 A/B 테스트가 가능합니다. 변수 간 상호작용 없이 개별 효과를 측정할 수 있습니다.

단점: 충돌 가능성

독립적이기 때문에 변수 간 모순이 발생할 수 있습니다. 예: DNA가 "relaxed" 포즈를 원하지만 Preset은 "power_stance"를 지정. 현재 이 충돌을 감지하는 로직이 없습니다.

연결 가능성

현재 4개 변수는 모두 독립적입니다. 하지만 연결하면 더 스마트해질 수 있습니다. 예를 들어: DNA Profile이 어떤 Preset 기본값을 오버라이드할지 직접 지시하거나, 이미지 분석(VLM)이 생성 결과가 DNA를 잘 반영했는지 검증하는 피드백 루프를 구성할 수 있습니다.

6.4 — 배경 인풋의 소속

배경 정보의 소스를 어디에 위치시킬 것인지에 대한 아키텍처 결정입니다. 이것은 6.1의 카테고리 정의 문제의 구체적 사례이기도 합니다 — 배경이 DNA에 포함되어야 하는가, Preset에 남아야 하는가?

Option A
배경을 Brand Preset에 넣기
Pros
  • 브랜드 톤과 일관성 보장
  • 관리 쉬움 (JSON 파일 하나)
  • 새 브랜드 추가 시 자동 배경
Cons
  • 유저별 커스텀 불가
  • 유연성 떨어짐
Current Status
  • preset.background.장소 / 배경상세 (이미 존재)
Option B
배경을 DNA Profile에 넣기
Pros
  • 유저 레퍼런스에서 학습
  • 더 세밀한 배경 DNA
Cons
  • 레퍼런스에 배경이 명확히 없으면 추출 어려움
  • 카테고리 추가 필요 (현재 pose/expression/vibe만)
Current Status
  • DNA에 배경 카테고리 없음
Option C
현재 구조 유지 (하이브리드)
Pros
  • Preset = 기본 배경
  • Background Ref = 유저 오버라이드
  • 기존 코드 변경 없음
Cons
  • 3곳 분산 (preset + VLM + 이미지)
  • 우선순위 불명확
  • 충돌 시 디버깅 어려움
Current Status
  • 현재 운영 중인 방식

Recommendation

Option A + C 혼합을 권장합니다. Brand Preset을 배경의 단일 소스 오브 트루스(Single Source of Truth)로 설정하되, Background Ref가 제공된 경우에만 명시적으로 오버라이드하는 구조입니다.

구체적으로는: (1) Preset의 background 필드가 기본값이 됩니다. (2) Background Ref 이미지가 strict 또는 loose 모드로 제공되면 Preset 배경을 완전 대체합니다. (3) VLM 배경 분석은 프롬프트 보강용으로만 사용하고, 최종 배경 결정 로직에서는 제외합니다.

이를 통해 배경 정보의 소스가 최대 2곳으로 줄어들고, 우선순위가 명확해집니다: Background Ref > Preset > (VLM은 보조).

Section 07

개선 제안

현재 파이프라인의 구조적 한계를 개선하기 위한 3가지 제안입니다.

01

DNA 영향력 확대

adapt_prompt()가 패치하는 필드를 확대하여 DNA Profile의 영향력을 현재 ~15%에서 ~25%로 증가시킵니다.

추가 패치 대상

  • 다리 포즈 (legs position)
  • 힙 각도 (hip angle)
  • 눈썹 표정 (eyebrow expression)
  • 배경 밝기 톤 (background brightness)
  • 전체 색감 따뜻함/차가움 (color temperature)
02

변수 간 연결

DNA가 Preset을 참조하여 충돌을 감지하고, 모순되는 지시를 자동 조정하는 레이어를 추가합니다.

충돌 감지 예시

  • Preset="power_stance" + DNA="relaxed" → DNA 우선 또는 블렌드
  • Preset="smile" + DNA="neutral" → 가중 평균 적용
  • 배경 Preset="studio" + Bg Ref="outdoor" → Bg Ref 우선
03

배경 통합

배경 결정 로직을 단일 함수 resolve_background()로 통합하여 3곳에 분산된 배경 정보를 하나의 파이프라인으로 정리합니다.

통합 우선순위

  • 1순위: Background Ref 이미지 (strict/loose)
  • 2순위: Brand Preset background 필드
  • 3순위: VLM 배경 분석 (보조 텍스트로만)
  • Fallback: "clean studio background"
